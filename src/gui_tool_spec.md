# POファイルエディタ GUI仕様書

## 1. 現状の機能分析
既存スクリプトで提供されている機能：
- `import_unknown.py`: 未知の翻訳をPOTファイルにインポート
- `import_pot.py`: POTファイルの内容をPOファイルにインポート
- `import_mismatch.py`: 不一致の翻訳をPOTファイルにインポート
- `format_po_files.py`: POファイルのフォーマット整形
- `delete_extracted_comments.py`: 抽出コメントの削除

## 2. 基本要件
### 2.1 主要機能要件
1. ファイル管理
   - リポジトリ内のPOT/POファイルの自動検出と表示
   - ファイルダイアログによる手動ファイル選択機能
   - 選択したファイルパスの表示と編集機能
   - Unknown/Mismatchファイルの自動検出
     * POファイルと同じディレクトリから検出
     * バージョン番号（例：24_1, 25_1）に対応
     * 複数バージョンが存在する場合は最新を選択

2. 操作機能
   - 既存の5つのスクリプト機能をGUIから実行可能
   - 各操作の実行前に対象ファイルの確認と変更が可能
   - 処理結果のフィードバック表示
   - 処理の並行実行防止（他の処理実行中は新規実行をブロック）

### 2.2 UI要件
1. メインウィンドウ
   - タブ形式のインターフェース
     * ファイル設定タブ
     * 操作タブ
   - ステータス表示エリア（共通）
   - 最小ウィンドウサイズ: 800x600

2. ファイル設定タブ
   - POT/POファイルの一覧表示
   - ファイル選択ダイアログの提供
   - 選択したファイルパスの表示と編集
   - POファイルのリスト管理機能
     * 言語ごとのPOファイル設定
     * 言語の追加・削除機能（削除時は確認ダイアログ表示）
     * 各言語のファイルパス設定
     * リスト形式での表示と管理（スクロール可能）

3. 操作タブ
   - 各機能に対応するボタン
     * 未知の翻訳インポート
     * POTファイルインポート
     * 不一致翻訳インポート
     * POファイルフォーマット
     * 抽出コメント削除
   - 処理対象ファイルの確認ダイアログ
   - 処理進捗の表示

## 4. 技術仕様
### 4.1 開発環境
- 言語: Python
- GUIフレームワーク: PySide6（Qt for Python）
- 既存の`sgpo`ライブラリを継続利用
- 型アノテーションの活用

### 4.2 クラス構成
1. `MainWindow`
   - GUIのメインウィンドウを管理
   - タブウィジェットの管理
   - 各コンポーネントの配置と初期化
   - ファイル検出と管理機能
   - 操作実行の制御

2. `FileSettingsTab`
   - ファイル設定タブの管理
   - POTファイル設定
   - POファイルリスト管理
   - Unknown/Mismatchファイル設定

3. `OperationsTab`
   - 操作タブの管理
   - 各種操作ボタンの配置
   - 操作実行の制御

4. `LanguageFileRow`
   - 言語ごとのPOファイル設定行を管理
   - ファイル選択機能
   - 削除機能（確認ダイアログ付き）

5. `WorkerThread`
   - 非同期タスク実行
   - 標準出力のキャプチャ
   - 処理状態の通知（Signal経由）

## 5. 処理フロー
1. アプリケーション起動
   - リポジトリ内のPOT/POファイルを自動検出
   - POファイルと同じディレクトリからUnknown/Mismatchファイルを検出
     * バージョン番号パターン（[0-9]*_[0-9]*）に基づく検索
     * 最新バージョンの自動選択
   - 検出されたファイルパスを各入力欄に表示
   - POファイルリストを構築

2. POファイル管理
   - 言語の追加
     * [言語追加]ボタンクリック
     * 言語選択ダイアログ表示（TODO）
     * 選択された言語用の入力欄を追加
   - 言語の削除
     * 削除ボタンクリック
     * 確認ダイアログ表示
     * 承認後、該当言語の設定を削除

3. タスク実行
   - 操作ボタンクリック
   - 他の処理実行中でないことを確認
   - WorkerThreadでタスクを非同期実行
   - 処理状況をログエリアに表示
   - 完了時に結果をダイアログで通知

## 6. エラーハンドリング
- ファイル未選択時の警告
- ファイル不存在時のエラー表示
- 処理失敗時のエラーメッセージ表示
- 不正なファイル形式の検出と警告
- 他の処理実行中の重複実行防止
